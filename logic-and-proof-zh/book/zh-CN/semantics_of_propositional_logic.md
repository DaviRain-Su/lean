# 第 6 章：命题逻辑的语义

> 本译文对应原书 [Semantics of Propositional Logic](https://leanprover.github.io/logic_and_proof/semantics_of_propositional_logic.html)；英文 Sphinx 源：`semantics_of_propositional_logic.rst`。

经典地，我们认为命题变量的取值范围是那些可以为真或为假的陈述。直观上，我们认为一个证明系统告诉我们哪些命题公式**必须**为真，无论变量代表什么。例如，我们能够从假设 $A$、$B$ 和 $A \land B \to C$ 证明 $C$，这似乎告诉我们：只要这些假设为真，$C$ 也必然为真。

要理解这一点，需要跳出系统之外，给出关于真的说明——更精确地说，是一个命题公式在什么条件下为真。这正是符号逻辑被设计来做的事情之一，这项任务属于**语义学**（semantics）的范畴。公式和形式证明是**句法**概念，也就是说，它们由符号和符号结构表示。真则是一个**语义**概念，因为它把某种**意义**赋予某些公式。

句法上，我们能够提出并回答如下问题：

- 给定一组假设 $\Gamma$ 和一个公式 $A$，我们能否从 $\Gamma$ 推导出 $A$？
- 哪些公式可以从 $\Gamma$ 推导出来？
- 推导 $A$ 需要哪些假设？

我们语义上考虑的问题则不同：

- 给定对公式 $A$ 中出现的命题变量的一种真值赋值，$A$ 为真还是为假？
- 是否存在使 $A$ 为真的真值赋值？
- 哪些真值赋值使 $A$ 为真？

在本章中，我们不会给出句法和语义的完全严格的数学处理。这一主题更适合一门更高级、更专注于数理逻辑的课程。但我们会足够详细地讨论语义问题，让你对“语义地思考”意味着什么有一个良好的感觉，并了解如何实际利用语义概念。

## 6.1. 真值与赋值

我们需要的第一个概念是**真值**（truth value）。我们已经见过两个：即“真”和“假”。在非正式数学中，我们用符号 $\mathbf{T}$ 和 $\mathbf{F}$ 表示它们。这些就是自然演绎中 $\top$ 和 $\bot$ 以及 Lean 中 `True` 和 `False` 所要表示的值。

在本文中，我们将采用“经典”的真概念，遵循[第 5 章](classical_reasoning.md)的讨论。这可以以多种方式理解，但具体而言，它归结为：我们假设任何命题要么为真，要么为假（当然，不会同时为真又为假）。这种真概念是排中律 $A \lor \neg A$ 的基础。语义上，我们把这句话读作“要么 $A$ 为真，要么 $\neg A$ 为真”。由于在我们的语义解释中，$\neg A$ 为真当且仅当 $A$ 为假，排中律就是说 $A$ 要么为真要么为假。

我们需要的下一个概念是**真值赋值**（truth assignment），它就是一个给一组命题变量中的每个元素指派真值的函数。本节中，我们用字母 $P, Q, R, \ldots$ 表示命题变量，用 $A, B, C, \ldots$ 表示任意公式，以区分两者。例如，如下定义的函数 $v$

- $v(P) := \mathbf{T}$
- $v(Q) := \mathbf{F}$
- $v(R) := \mathbf{F}$
- $v(S) := \mathbf{T}$

就是变量集合 $\{ P, Q, R, S \}$ 的一个真值赋值。

直观上，真值赋值描述了一种可能的“世界状态”。回到“恶意与爱丽丝”谜题，假设以下字母是这些陈述的简写：

- $P$ := 爱丽丝的兄弟是受害者
- $Q$ := 爱丽丝是凶手
- $R$ := 爱丽丝在酒吧

在谜题解所描述的世界中，第一个和第三个陈述为真，第二个为假。因此我们的真值赋值给 $P$ 和 $R$ 赋值为 $\mathbf{T}$，给 $Q$ 赋值为 $\mathbf{F}$。

一旦我们有了对一组命题变量的真值赋值 $v$，就可以把它扩展为一个**赋值函数** $\bar{v}$，该函数给每个只依赖于这些变量的命题公式指派真或假。函数 $\bar{v}$ 是递归定义的，也就是说，公式自下而上求值：复合公式的值由其组成部分的值决定。形式上，定义如下：

- $\bar{v}(\top) = \mathbf{T}$。
- $\bar{v}(\bot) = \mathbf{F}$。
- $\bar{v}(\ell) = v(\ell)$，其中 $\ell$ 是任意命题变量。
- $\bar{v}(\neg A) = \mathbf{T}$ 当 $\bar{v}(A)$ 为 $\mathbf{F}$，反之亦然。
- $\bar{v}(A \land B) = \mathbf{T}$ 当 $\bar{v}(A)$ 和 $\bar{v}(B)$ 都是 $\mathbf{T}$，否则为 $\mathbf{F}$。
- $\bar{v}(A \lor B) = \mathbf{T}$ 当 $\bar{v}(A)$ 和 $\bar{v}(B)$ 中至少一个为 $\mathbf{T}$；否则为 $\mathbf{F}$。
- $\bar{v}(A \to B) = \mathbf{T}$ 当要么 $\bar{v}(B)$ 为 $\mathbf{T}$，要么 $\bar{v}(A)$ 为 $\mathbf{F}$，否则为 $\mathbf{F}$。（等价地，$\bar{v}(A \to B) = \mathbf{F}$ 当 $\bar{v}(A)$ 为 $\mathbf{T}$ 且 $\bar{v}(B)$ 为 $\mathbf{F}$，否则为 $\mathbf{T}$。）

合取和析取的规则很容易理解。“$A$ 并且 $B$”恰好在 $A$ 和 $B$ 都为真时为真；“$A$ 或 $B$”在 $A$、$B$ 至少一个为真时为真。

蕴涵的语义规则更难理解。人们常常惊讶地听说，任何前件为假的 if-then 陈述都应该为真。“如果我有两个头，那么圆就是方的”这句话听起来似乎应该是假的，但按我们的计算，它为真。要理解这一点，请思考下面两句话的区别：

- “如果我有两个头，那么圆就是方的。”
- “如果我有两个头，那么圆就会是方的。”

第二句话是**反事实**蕴涵（counterfactual implication）的例子。它断言如果世界与实际情况不同，世界可能会如何变化。哲学家们研究反事实已经几个世纪，但数理逻辑关注的是第一句话，一种**实质蕴涵**（material implication）。实质断言的是世界现在是什么样，而不是世界可能是什么样。由于“我有两个头”是假的，所以“如果我有两个头，那么圆就是方的”为真。

为什么我们要这样评估实质蕴涵？让我们再次考虑真命题“每个是素数且大于 2 的自然数都是奇数”。我们可以把这个句子理解为下面这个无限列表中的所有句子都为真：

- 如果 0 是素数且大于 2，那么 0 是奇数。
- 如果 1 是素数且大于 2，那么 1 是奇数。
- 如果 2 是素数且大于 2，那么 2 是奇数。
- 如果 3 是素数且大于 2，那么 3 是奇数。
- ……

这个列表中的第一个句子很像我们的“两个头”例子，因为前件和后件都为假。但由于它是一般为真的陈述的一个实例，我们必须给它赋值为 $\mathbf{T}$。第二个句子不同：前件仍为假，但后件为真。这两者合起来告诉我们：当前件为假时，条件句应为真。第四个句子前件为真、后件为真。所以从第二和第四个句子我们看到：当后件为真时，条件句也应为真。最后，“如果 3 是素数且大于 2，那么 3 是偶数”显然**不**应为真。这种前件为真、后件为假的模式，是条件句唯一为假的情况。

让我们再用上一章描述的演绎规则来 motivate 实质蕴涵的语义。注意，如果 $B$ 为真，我们可以在没有任何关于 $A$ 的假设的情况下证明 $A \to B$：

![推理规则 1](https://leanprover-community.github.io/logic_and_proof/_static/semantics_of_propositional_logic.1.png)

这来自于对蕴涵引入规则的正确理解：给定 $B$，总可以推出 $A \to B$，然后取消假设 $A$——**如果存在的话**。如果 $A$ 在证明中从未被使用，结论只是比需要的更弱。这个推理在 Lean 中得到验证：

```lean
section
variable (A B : Prop) (hB : B)

example : A → B :=
fun hA : A ↦
  show B from hB

end
```

类似地，如果 $A$ 为假，我们可以在没有任何关于 $B$ 的假设的情况下证明 $A \to B$：

![推理规则 2](https://leanprover-community.github.io/logic_and_proof/_static/semantics_of_propositional_logic.2.png)

在 Lean 中：

```lean
section
variable (A B : Prop) (hnA : ¬ A)

example : A → B :=
fun hA : A ↦
  show B from False.elim (hnA hA)

end
```

最后，如果 $A$ 为真而 $B$ 为假，我们可以证明 $\neg (A \to B)$：

![推理规则 3](https://leanprover-community.github.io/logic_and_proof/_static/semantics_of_propositional_logic.3.png)

同样，在 Lean 中：

```lean
section
variable (A B : Prop) (hA : A) (hnB : ¬B)

example : ¬ (A → B) :=
fun h : A → B ↦
  have hB : B := h hA
  show False from hnB hB

end
```

现在我们已经定义了公式相对于真值赋值的真假，就可以回答第一个语义问题了：给定对公式 $\varphi$ 中出现的命题变量的一个赋值 $v$，如何判断 $\varphi$ 是否为真？这等价于计算 $\bar{v}(\varphi)$，而 $\varphi$ 的递归定义给出了方法：我们从下往上求值 $\varphi$ 中出现的表达式，从命题变量开始，利用各组成部分的求值结果来求整个表达式的值。例如，假设真值赋值 $v$ 使 $A$ 和 $B$ 为真、$C$ 为假。要在 $v$ 下求值 $(B \to C) \lor (A \land B)$，注意 $B \to C$ 为假，$A \land B$ 为真。由于析取“假或真”为真，整个公式为真。

我们也可以朝另一个方向走：给定一个公式，尝试找到使它为真（或为假）的真值赋值。事实上，我们可以用 Lean 来为我们求值公式。在下面的例子中，你可以给命题符号 `A`、`B`、`C`、`D`、`E` 任意赋值。当你用这些输入运行 Lean 时，`eval` 语句的输出就是表达式的值。

```lean
section

-- 在这里定义你的真值赋值
def A := true
def B := false
def C := true
def D := true
def E := false

def test (p : Prop) [Decidable p] : String :=
if p then "true" else "false"

#eval test ((A ∧ B) ∨ ¬ C)
#eval test (A → D)
#eval test (C → (D ∨ ¬E))
#eval test (¬(A ∧ B ∧ C ∧ D))

end
```

试着改变真值赋值，看看会发生什么。你可以在输入末尾添加自己的公式并求值。试着找到使上面每个公式都为真的真值赋值。作为额外挑战，试着找到一个使它们同时为真的单一真值赋值。

## 6.2. 真值表

我们提出的第二和第三个语义问题比第一个稍微复杂一些。给定一个公式 $A$，是否存在使它为真的真值赋值？如果存在，哪些真值赋值使 $A$ 为真？这些问题不是考虑某个特定真值赋值，而是要求我们对**所有**可能的真值赋值进行量化。

当然，可能的真值赋值数量取决于考虑的命题字母数量。由于每个字母有两种可能取值，$n$ 个字母会产生 $2^{n}$ 种可能的真值赋值。这个数量增长非常快，所以我们这里主要考察较小的公式。

我们将使用一种称为**真值表**（truth table）的工具来判断一个公式何时（如果可能的话）为真。在真值表的左侧，我们列出当前命题字母的所有可能真值赋值；在右侧，我们列出整个公式在对应赋值下的真值。

首先，真值表可以用来简洁地总结我们的逻辑联结词的语义：

## 6.3. 可靠性与完全性

假设我们心中有一个固定的演绎系统，例如自然演绎。一个命题公式如果在该系统中有形式证明，则称它是**可证的**（provable）。一个命题公式如果在任何真值赋值下都为真，则称它是**重言式**（tautology）或**有效的**（valid）。可证性是句法概念，因为它断言一种句法对象——即证明——的存在。有效性是语义概念，因为它与真值赋值和赋值函数有关。但直观上，这两个概念应当一致：两者都表达了公式 $A$ **必然**为真的思想，人们也会期望一个好的证明系统能够让我们推导出所有有效公式。

“每个可证公式都是有效的”这一陈述称为证明系统相对于基于赋值的语义的**可靠性**（soundness）。如果 $A$ 是任意公式，逻辑学家用记号 $\vdash A$ 表示 $A$ 可证，用记号 $\vDash A$ 表示 $A$ 有效。（第一个符号有时称为“ turnstile”，第二个称为“double-turnstile”。）依此记号，可靠性说：对每个命题公式 $A$，如果 $\vdash A$，那么 $\vDash A$。其逆命题，即每个有效公式都是可证的，称为**完全性**（completeness）（同样：证明系统相对于基于赋值的语义的完全性）。用符号表示，它说：对每个公式 $A$，如果 $\vDash A$，那么 $\vdash A$。

由于我们选择推理规则和定义赋值函数的方式，这种两个概念应当一致的直觉确实成立。换言之，我们为命题逻辑给出的自然演绎系统相对于真值表语义是可靠且完全的。

可靠性和完全性的概念可以扩展到从假设出发的可证性。如果 $\Gamma$ 是一组命题公式，$A$ 是一个命题公式，那么如果对任何使 $\Gamma$ 中每个公式都为真的真值赋值，$A$ 也都为真，则称 $A$ 是 $\Gamma$ 的**逻辑后承**（logical consequence）。在这个扩展设定中，可靠性说：如果 $A$ 可从 $\Gamma$ 证明，那么 $A$ 是 $\Gamma$ 的逻辑后承。完全性则朝另一个方向：如果 $A$ 是 $\Gamma$ 的逻辑后承，那么 $A$ 可从 $\Gamma$ 证明。用符号表示，我们写 $\Gamma \vdash A$ 表示 $A$ 可从 $\Gamma$ 中的公式证明（或 $\Gamma$ **证明** $A$），写 $\Gamma \vDash A$ 表示 $A$ 是 $\Gamma$ 的逻辑后承（或 $\Gamma$ **蕴涵** $A$）。依此记号，可靠性说：对每个命题公式 $A$ 和命题公式集合 $\Gamma$，如果 $\Gamma \vdash A$ 那么 $\Gamma \vDash A$；完全性说：对每个 $A$ 和 $\Gamma$，如果 $\Gamma \vDash A$ 那么 $\Gamma \vdash A$。

给定一组命题公式 $\Gamma$ 和一个命题公式 $A$，上一节给了我们判断 $\Gamma$ 是否蕴涵 $A$ 的方法：为 $\Gamma$ 和 $A$ 中的所有公式构造真值表，检查在每一行上，当 $\Gamma$ 中每个公式都为真时，$A$ 是否也为真。（在 $\Gamma$ 中某个公式为假的那些行上，$A$ 的值无关紧要。）

注意，根据自然演绎的规则，公式 $A$ 可从假设集合 $\{ B_{1}, B_{2}, \ldots, B_{n} \}$ 证明，当且仅当公式 $B_{1} \land B_{2} \land \ldots \land B_{n} \to A$ 可以无需任何假设而证明。因此，至少对于有限的公式集合 $\Gamma$，可靠性和完全性的两种表述是等价的。

证明可靠性和完全性属于**元理论**（metatheory）的范畴，因为它要求我们关于我们的推理方法进行推理。这不是本书的中心焦点：我们更关心**使用**逻辑和真的概念，而不是建立它们的性质。但可靠性和完全性的概念在帮助我们理解逻辑概念的本质方面起着重要作用，因此我们将在这里尝试给出一些关于为什么这些性质对命题逻辑成立提示。

证明可靠性比证明完全性更容易。我们希望证明：每当 $A$ 可从假设集 $\Gamma$ 证明时，$A$ 就是 $\Gamma$ 的逻辑后承。在后面的章节中，我们将考虑数学归纳法证明，它使我们能够通过证明某个性质对一些“简单”对象成立，并且在过渡到更复杂对象时保持，从而确立该性质对一般对象集合成立。对于自然演绎，只需证明可靠性对最基本的证明——使用假设规则——成立，并且它在每条推理规则下保持。基本情况很容易：假设规则说 $A$ 可从假设 $A$ 证明，显然任何使 $A$ 为真的真值赋值也使 $A$ 为真。归纳步骤也不难；它们涉及检查我们所选择的规则是否与语义概念相契合。例如，假设最后一条规则是合取引入规则。在这种情况下，我们有从某些假设 $\Gamma$ 证明 $A$，以及从某些假设 $\Delta$ 证明 $B$，我们将它们组合成从 $\Gamma \cup \Delta$（即两者中的假设）证明 $A \land B$。归纳地，我们可以假设 $A$ 是 $\Gamma$ 的逻辑后承，$B$ 是 $\Delta$ 的逻辑后承。设 $v$ 是使 $\Gamma \cup \Delta$ 中每个公式都为真的任意真值赋值。那么根据归纳假设，它使 $A$ 为真，也使 $B$ 为真。根据赋值函数的定义，$\bar{v}(A \land B) = \mathbf{T}$，证毕。

证明完全性更难。只需证明：如果 $A$ 是任意重言式，那么 $A$ 可证。一种策略是证明自然演绎可以模拟真值表方法。例如，假设 $A$ 由命题变量 $B$ 和 $C$ 构成。那么在自然演绎中，我们应该能够证明

$$
(B \land C) \lor (B \land \neg C) \lor (\neg B \land C) \lor (\neg B \land \neg C) ,
$$

每个析取支对应真值表的一行。然后，我们应该能够利用每个析取支来“求值”公式 $A$ 的每个子公式，按照其赋值证明它为真或为假，直到我们得到 $A$ 本身的证明。

一种更漂亮的方式是以允许我们从 $A$ 向后工作寻找证明的形式表达自然演绎规则。换言之，首先，我们给出一个通过从 $A$ 向后工作来构造 $A$ 的推导的过程。然后我们论证：如果该过程失败，那么在失败点我们可以找到一个使 $A$ 为假的真值赋值。结果，如果每个真值赋值都使 $A$ 为真，该过程就会返回 $A$ 的证明。
