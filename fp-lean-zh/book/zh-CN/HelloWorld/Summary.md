# 求值与执行

副作用是程序执行中超出数学表达式求值范畴的方面，例如读取文件、抛出异常或触发工业机械。
大多数语言允许在求值过程中发生副作用，但 Lean 不允许。
相反，Lean 有一个名为 {moduleName}`IO` 的类型，它表示使用副作用的程序的_描述_。
这些描述由语言的运行时系统执行，运行时系统会调用 Lean 表达式求值器来完成特定的计算。
类型为 {moduleTerm}`IO α` 的值称为 _{moduleName}`IO` 动作_。
最简单的是 {moduleName}`pure`，它返回其参数且没有实际的副作用。

{moduleName}`IO` 动作也可以被理解为以整个世界作为参数并返回副作用已发生的新世界的函数。
在底层，{moduleName}`IO` 库确保世界永远不会被复制、创建或销毁。
虽然这种副作用模型无法真正实现，因为整个宇宙太大而无法放入内存，但现实世界可以通过一个在程序中传递的标记来表示。

名为 {anchorName MainTypes}`main` 的 {moduleName}`IO` 动作在程序启动时执行。
{anchorName MainTypes}`main` 可以有三种类型之一：
 * {anchorTerm MainTypes}`main : IO Unit` 用于无法读取命令行参数且始终返回退出码 {anchorTerm MainTypes}`0` 的简单程序，
 * {anchorTerm MainTypes}`main : IO UInt32` 用于不带参数但可能表示成功或失败的程序，以及
 * {anchorTerm MainTypes}`main : List String → IO UInt32` 用于接受命令行参数并表示成功或失败的程序。


# {lit}`do` 记法

Lean 标准库提供了许多基本的 {moduleName}`IO` 动作，它们表示诸如从文件读写、与标准输入和标准输出交互等效果。
这些基础的 {moduleName}`IO` 动作使用 {kw}`do` 记法组合成更大的 {moduleName}`IO` 动作，{kw}`do` 记法是一种用于编写带副作用程序描述的内置领域特定语言。
{kw}`do` 表达式包含一系列_语句_，它们可以是：
 * 表示 {moduleName}`IO` 动作的表达式，
 * 使用 {kw}`let` 和 {lit}`:=` 的普通局部定义，其中定义的名称指代所提供表达式的值，或者
 * 使用 {kw}`let` 和 {lit}`←` 的局部定义，其中定义的名称指代所提供表达式的执行结果。

用 {kw}`do` 编写的 {moduleName}`IO` 动作一次执行一条语句。

此外，出现在 {kw}`do` 正下方的 {kw}`if` 和 {kw}`match` 表达式会被隐式视为在每个分支中都有自己的 {kw}`do`。
在 {kw}`do` 表达式内部，_嵌套动作_是指在圆括号下紧接左箭头的表达式。
Lean 编译器会隐式将它们提升到最近的封闭 {kw}`do`（这可能是 {kw}`match` 或 {kw}`if` 表达式的分支的隐式 {kw}`do`），并赋予它们一个唯一的名称。
然后这个唯一的名称会替换嵌套动作的原始位置。


# 编译与运行程序

由单个文件组成并带有 {moduleName}`main` 定义的 Lean 程序可以使用 {lit}`lean --run FILE` 运行。
虽然这是开始编写简单程序的好方法，但大多数程序最终都会发展为多文件项目，需要在运行前进行编译。

Lean 项目被组织成_包_，包是库和可执行文件的集合，同时包含依赖信息和构建配置。
包使用 Lake（一个 Lean 构建工具）来描述。
使用 {lit}`lake new` 在新目录中创建 Lake 包，或使用 {lit}`lake init` 在当前目录中创建。
Lake 包配置是另一种领域特定语言。
使用 {lit}`lake build` 构建项目。


# 偏性

遵循表达式求值的数学模型的一个结果是，每个表达式都必须有一个值。
这就排除了未能覆盖数据类型所有构造子的不完整模式匹配，以及可能陷入无限循环的程序。
Lean 确保所有 {kw}`match` 表达式覆盖所有情况，并确保所有递归函数要么是结构递归的，要么具有显式的终止证明。

然而，一些真实程序需要无限循环的可能性，因为它们处理潜在无限的数据，例如 POSIX 流。
Lean 提供了一个逃生通道：定义中标记为 {kw}`partial` 的函数不要求终止。
但这有代价。
由于类型是 Lean 语言的一等组成部分，函数可以返回类型。
然而，偏函数不会在类型检查期间求值，因为函数中的无限循环可能导致类型检查器进入无限循环。
此外，数学证明无法检查偏函数的定义，这意味着使用它们的程序不太适合形式化证明。
