# 类型类与重载

%%%
tag := none
%%%

类型类是 Lean 重载函数和运算符的机制。
多态函数可以与多种类型一起使用，但无论用于哪种类型，其行为方式都相同。
例如，一个拼接两个列表的多态函数，无论列表条目的类型是什么都可以使用，但它无法根据具体是哪种类型而有不同的行为。
另一方面，通过类型类重载的运算也可以与多种类型一起使用。
然而，每种类型都需要该重载运算自己的实现。
这意味着行为可以根据所提供的类型而变化。

**类型类（type class）** 有一个名称、若干参数，以及由若干带类型的名称组成的体。
名称是引用重载运算的方式，参数决定定义的哪些方面可以重载，体提供可重载运算的名称和类型签名。
每个可重载运算称为类型类的一个 {deftech}_方法（method）_。
类型类可以为某些方法提供相对于其他方法的默认实现，在不需要时使实现者不必手工定义每个重载。

类型类的 {deftech}_实例（instance）_ 为给定参数提供方法的实现。
实例可以是多态的，此时它们可以适用于多种参数；它们也可以为某些特定类型提供更高效的默认方法的具体实现。

类型类参数要么是 {deftech}_输入参数（input parameters）_（默认），要么是 {deftech}_输出参数（output parameters）_（由 {moduleName}`outParam` 修饰符标明）。
Lean 在所有输入参数都不再是元变量之前不会开始搜索实例，而输出参数可以在搜索实例的过程中被求解。
类型类的参数不必是类型——它们也可以是普通值。
用于重载自然数字面量的 {moduleName}`OfNat` 类型类，把被重载的 {moduleName}`Nat` 本身作为参数，这允许实例限制允许的数字。

实例可以用 {anchorTerm defaultAdd}`@[default_instance]` 属性标记。
当实例是默认实例时，当 Lean 否则会因类型中存在元变量而找不到实例时，它会被选作后备。

# 常见语法的类型类

%%%
tag := none
%%%

Lean 中的大多数中缀运算符都通过类型类重载。
例如，加法运算符对应名为 {moduleName}`Add` 的类型类。
这些运算符大多数都有对应的异构版本，其中两个参数不必具有相同类型。
这些异构运算符通过名称以 {lit}`H` 开头的类版本来重载，例如 {moduleName}`HAdd`。

索引语法通过名为 {moduleName}`GetElem` 的类型类重载，它涉及证明。
{moduleName}`GetElem` 有两个输出参数，分别是要从集合中提取的元素的类型，以及可用于确定什么算作索引值在集合边界内的证据的函数。
这一证据由命题描述，当使用数组索引时 Lean 会尝试证明该命题。
当 Lean 无法在编译时检查列表或数组访问操作是否在边界内时，可以通过在索引语法后附加 {lit}`?` 把检查推迟到运行时。

# 函子

%%%
tag := none
%%%


函子是一种支持映射运算的多态类型。
这一映射运算会“就地”变换所有元素，而不改变其他结构。
例如，列表是函子，映射运算既不能丢弃、复制，也不能打乱列表中的条目。

虽然函子由拥有 {anchorName FunctorDef}`map` 来定义，但 Lean 中的 {anchorName FunctorDef}`Functor` 类型类包含一个额外的默认方法，负责把常数函数映射过某个值，把所有由多态类型变量给出类型的值替换为同一个新值。
对于某些函子，这比遍历整个结构更高效。

# 推导实例

%%%
tag := none
%%%


许多类型类有非常标准的实现。
例如，布尔相等类 {moduleName}`BEq` 通常通过先检查两个参数是否用同一构造子构造，再检查它们的所有参数是否相等来实现。
这些类的实例可以 _自动_ 创建。

在定义归纳类型或结构体时，声明末尾的 {kw}`deriving` 子句会使实例被自动创建。
此外，{kw}`deriving instance`﻿{lit}` ... `﻿{kw}`for`﻿{lit}` ...` 命令可以在数据类型定义之外使用，以生成实例。
由于每个可推导实例的类都需要特殊处理，并非所有类都可推导。

# 强制转换

%%%
tag := none
%%%


强制转换允许 Lean 通过插入把数据从一种类型变换为另一种类型的函数调用，从原本会是编译时错误的情况中恢复。
例如，从任意类型 {anchorName CoeOption}`α` 到类型 {anchorTerm CoeOption}`Option α` 的强制转换，允许直接写值，而不使用 {anchorName CoeOption}`some` 构造子，使 {anchorName CoeOption}`Option` 更像面向对象语言中的可空类型。

强制转换有多种。
它们可以从不同种类的错误中恢复，并由各自的类型类表示。
{anchorName CoeOption}`Coe` 类用于从类型错误中恢复。
当 Lean 在期望类型为 {anchorName Coe}`β` 的上下文中遇到类型为 {anchorName Coe}`α` 的表达式时，Lean 首先尝试串起一条可以把 {anchorName Coe}`α` 变换为 {anchorName Coe}`β` 的强制转换链，只有在这做不到时才显示错误。
{moduleName}`CoeDep` 类把被强制转换的具体值作为额外参数，允许对该值进行进一步的类型类搜索，或允许在实例中使用构造子来限制转换的范围。
{moduleName}`CoeFun` 类拦截在编译函数应用时原本会出现的“不是函数”错误，并在可能时允许把函数位置上的值变换为实际的函数。
