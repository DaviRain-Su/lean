# 小结

# 编码副作用

Lean 是一门纯函数式语言。这意味着它不包含可变变量、日志记录或异常等副作用。不过，大多数副作用都可以使用函数与归纳类型或结构体的组合来_编码_。例如，可变状态可以编码为从初始状态到最终状态与结果之配对的函数，异常则可以编码为带有表示成功终止与错误的构造子的归纳类型。

每一组编码后的效果都是一个类型。因此，如果程序使用了这些编码后的效果，这一点会体现在其类型中。函数式编程并不意味着程序不能使用效果，它只是要求程序对自己使用哪些效果保持_诚实_。Lean 的类型签名不仅描述函数期望的参数类型及其返回的结果类型，还描述它可能使用哪些效果。

# 单子类型类

即使在允许随处产生效果的语言中，也可以编写纯函数式程序。例如，{python}`2 + 3` 是一个完全没有效果的合法 Python 程序。类似地，组合具有效果的程序需要一种方式来规定效果必须发生的顺序。毕竟，异常究竟是在修改变量之前还是之后抛出，这一点很重要。

类型类 {anchorName FakeMonad}`Monad` 抓住了这两个重要性质。它有两个方法：{anchorName FakeMonad}`pure` 表示没有副作用的程序，{anchorName FakeMonad}`bind` 则用于串联具有效果的程序。{anchorName FakeMonad}`Monad` 实例的约定确保 {anchorName FakeMonad}`bind` 和 {anchorName FakeMonad}`pure` 确实分别刻画了纯计算与串联。

# 单子的 {kw}`do` 记法

{kw}`do` 记法并不局限于 {moduleName}`IO`，它对任何单子都适用。它允许使用单子的程序以一种让人联想到面向语句的语言风格来编写，语句一个接一个地串联。此外，{kw}`do` 记法还支持许多额外的便捷简写，例如嵌套动作。用 {kw}`do` 编写的程序在幕后会被翻译成对 {lit}`>>=` 的应用。

# 自定义单子

不同语言提供不同的副作用集合。虽然大多数语言都有可变变量和文件 I/O，但并非所有语言都具备异常等特性。其他语言则提供罕见或独有的效果，例如 Icon 的基于搜索的程序执行、Scheme 和 Ruby 的延续（continuations），以及 Common Lisp 的可恢复异常。用单子编码效果的一个优势在于，程序并不局限于语言本身提供的效果集合。由于 Lean 的设计目标是让使用任何单子进行编程都变得方便，程序员可以自由地为任何给定应用选择恰好合适的那组副作用。

# {lit}`IO` 单子

能够影响真实世界的程序在 Lean 中写作 {moduleName}`IO` 动作。{moduleName}`IO` 是众多单子之一。{moduleName}`IO` 单子编码了状态与异常，其中状态用于跟踪世界的状态，异常则用来建模失败与恢复。
