# 求值表达式

作为学习 Lean 的程序员，最需要理解的就是**求值（evaluation）**是如何工作的。求值就是找出一个表达式的值，就像我们在算术中所做的那样。例如，$15 - 6$ 的值是 $9$，而 $2 \times (3 + 1)$ 的值是 $8$。求后者的值时，先把 $3 + 1$ 替换成 $4$，得到 $2 \times 4$，再进一步归约为 $8$。有时候，数学表达式中会包含变量：在知道 $x$ 的值之前，我们无法计算 $x + 1$ 的值。在 Lean 中，程序首先就是表达式，而理解计算的主要方式就是把这些表达式求值，得到它们的值。

大多数编程语言都是**命令式（imperative）**的：一个程序由一系列按顺序执行的语句组成，通过这些语句的执行来得到程序的结果。程序可以访问可变内存，因此一个变量所指向的值可能随时间变化。除了可变状态之外，程序还可能有其他**副作用（side effects）**，例如删除文件、建立网络连接、抛出或捕获异常、从数据库读取数据等。"副作用"本质上是一个统称，用来描述程序中那些不遵循数学表达式求值模型的事情。

然而，在 Lean 中，程序的工作方式与数学表达式相同。变量一旦被赋值，就不能重新赋值。对一个表达式求值不会产生副作用。如果两个表达式的值相同，那么用其中一个替换另一个不会让程序算出不同的结果。这并不意味着 Lean 不能向控制台输出 `Hello, world!`，而是说执行输入/输出（I/O）并不是使用 Lean 的核心体验。因此，本章重点介绍如何在编辑器中与 Lean 交互式地求值表达式；下一章再介绍如何编写、编译并运行 `Hello, world!` 程序。

## 使用 `#eval`

要让 Lean 求值一个表达式，只需在编辑器中在该表达式前面写上 `#eval`，编辑器就会把结果反馈回来。通常，把光标或鼠标移到 `#eval` 上就能看到结果。例如：

```lean
#eval 1 + 2
```

得到值：

```
3
```

Lean 遵循算术运算符通常的优先级和结合性规则。也就是说：

```lean
#eval 1 + 2 * 5
```

得到值 `11`，而不是 `15`。

## 函数应用

普通的数学记号和大多数编程语言都使用括号来把函数应用到参数上（例如 `f(x)`），而 Lean 直接把函数写在参数旁边（例如 `f x`）。函数应用是最常见的操作之一，因此保持简洁很重要。例如，要计算 `"Hello, Lean!"`，我们不会写：

```lean
#eval String.append("Hello, ", "Lean!")
```

而是写成：

```lean
#eval String.append "Hello, " "Lean!"
```

函数的两个参数直接写在它旁边，用空格隔开。

正如算术的运算顺序规则要求表达式 `(1 + 2) * 5` 中使用括号一样，当一个函数的参数本身需要通过另一个函数调用来计算时，也需要括号。例如：

```lean
#eval String.append "great " (String.append "oak " "tree")
```

这里必须加括号，否则第二个 `String.append` 会被解释成第一个 `String.append` 的参数，而不是一个接收 `"oak "` 和 `"tree"` 作为参数的函数。先计算内层 `String.append` 调用的值，再把它追加到 `"great "` 上，最终得到 `"great oak tree"`。

## 条件表达式

命令式语言通常有两种条件结构：一种是**条件语句**，根据布尔值决定执行哪些指令；另一种是**条件表达式**，根据布尔值决定求值两个表达式中的哪一个。例如，在 C 和 C++ 中，条件语句用 `if` 和 `else` 编写，而条件表达式用三元运算符 `?` 和 `:` 把条件与分支分开。在 Python 中，条件语句以 `if` 开头，而条件表达式把 `if` 放在中间。

因为 Lean 是一门面向表达式的函数式语言，所以没有条件语句，只有条件表达式。它们使用 `if`、`then` 和 `else` 编写。例如：

```lean
String.append "it is " (if 1 > 2 then "yes" else "no")
```

求值过程如下：

```
String.append "it is " (if 1 > 2 then "yes" else "no")
===>
String.append "it is " (if false then "yes" else "no")
===>
String.append "it is " "no"
===>
"it is no"
```

为了简洁，一连串这样的求值步骤有时会用箭头连接表示。

## 你可能遇到的信息

如果让 Lean 求值一个缺少参数的函数应用，就会产生错误信息。例如：

```lean
#eval String.append "it is "
```

会给出一条相当长的错误信息：

```
could not synthesize a `ToExpr`, `Repr`, or `ToString` instance for type
  String → String
```

这条信息出现的原因是：Lean 中只应用了部分参数的函数会返回一个新的函数，等待剩余参数。Lean 无法向用户显示函数，因此在被要求这样做时会返回错误。
