# 类型

**类型（type）**根据程序能计算出的值来对程序进行分类。类型在程序中扮演多种角色：

1. 它们让编译器能够决定一个值在内存中的表示方式。
2. 它们帮助程序员向他人传达意图，充当函数输入和输出的轻量级规范；编译器会确保程序符合这一规范。
3. 它们能够防止各种潜在错误，例如把数字和字符串相加，从而减少程序所需的测试数量。
4. 它们帮助 Lean 编译器自动生成辅助代码，避免样板代码。

Lean 的类型系统 unusually expressive（具有极强的表达能力）。类型可以编码很强的规范，例如“这个排序函数返回其输入的一个排列”；也可以编码灵活的规范，例如“这个函数根据参数值的不同返回不同类型”。类型系统甚至可以被用作完整的逻辑体系来证明数学定理。然而，这种前沿的表达力并不意味着简单类型就不重要；理解这些简单类型是使用更高级特性的先决条件。

## 每个表达式都必须有类型

Lean 中的每个程序都必须有类型。特别地，每个表达式在被求值之前都必须有类型。到目前为止的例子中，Lean 都能够自动推断出类型，但有时候必须显式提供类型。这通过在括号内使用冒号操作符来完成：

```lean
#eval (1 + 2 : Nat)
```

这里，`Nat` 是**自然数（natural number）**的类型，即任意精度的非负整数。在 Lean 中，`Nat` 是非负整数字面量的默认类型。这个默认类型并不总是最佳选择。在 C 中，当减法结果会小于零时，无符号整数会下溢到最大可表示数。然而，`Nat` 可以表示任意大的非负整数，因此不存在“最大的数”可以下溢到。所以，`Nat` 上的减法在结果本会为负时返回 `0`。例如：

```lean
#eval (1 - 2 : Nat)
```

求值结果是 `0`，而不是 `-1`。如果要使用能表示负整数的类型，可以直接指定：

```lean
#eval (1 - 2 : Int)
```

使用这个类型后，结果就如预期是 `-1`。

## 使用 `#check`

如果只想检查一个表达式的类型而不求值它，可以使用 `#check` 代替 `#eval`。例如：

```lean
#check (1 - 2 : Int)
```

会报告 `1 - 2 : Int`，而不实际执行减法。

## 类型错误

当一个程序无法被赋予类型时，`#check` 和 `#eval` 都会返回错误。例如：

```lean
#check String.append ["hello", " "] "world"
```

会输出：

```
Application type mismatch: The argument
  ["hello", " "]
has type
  List String
but is expected to have type
  String
in the application
  String.append ["hello", " "]
```

这是因为 `String.append` 的第一个参数预期是一个字符串，但这里却提供了一个字符串列表。
