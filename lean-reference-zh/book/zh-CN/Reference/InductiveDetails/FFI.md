# 归纳类型与 FFI

> 对应英文：[FFI](https://lean-lang.org/doc/reference/latest/The-Type-System/Inductive-Types/#inductive-types-ffi)，抓取日期：2026-06-16。

当归纳类型要与外部语言或运行时接口交互时，仅理解其逻辑定义还不够；还必须理解它在编译后如何表示。

## 为什么 FFI 会在这里出现

FFI 关心的是：

- 这个 Lean 类型在内存里长什么样；
- proof 字段是否会被擦除；
- 是否是 trivial wrapper；
- 是否存在特殊运行时表示。

这些都直接依赖归纳类型的运行时布局，而不是只依赖高层 `inductive` 语法。

## 风险

若把一个逻辑上看似简单的归纳类型直接映射到外部接口，而不考虑：

- constructor tag
- 包装层
- 被擦除字段
- 内存布局

就可能得到错误甚至不安全的互操作。

## 使用建议

- 归纳类型参与 FFI 时，必须同时看逻辑模型和运行时表示。
- 若某类型需要稳定 FFI 布局，通常应显式设计并尽量保持简单。
- 不要把“Lean 源码层看起来像一个结构”直接等同于“FFI 层也就是那个结构体布局”。
