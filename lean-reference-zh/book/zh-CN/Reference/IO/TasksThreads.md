# 任务与线程

> 对应英文：[Tasks and Threads](https://lean-lang.org/doc/reference/latest/IO/Tasks-and-Threads/)，抓取日期：2026-06-16。

Lean 提供 task 和 thread 支持，用来表达并发和并行执行。`Task` API 允许把工作拆成可等待的单元，由 runtime 调度到底层线程运行。

## `Task` 的角色

`Task` 适合表示：

- 某个未来会产生结果的计算；
- 可与主线程并发执行的工作；
- 可组合、可等待、可在多个阶段汇总的并行子任务。

与“直接开 OS thread”相比，`Task` 更贴近 Lean 运行时提供的并发抽象，也更容易在纯函数式代码里表达依赖关系。

## 并发与同步

多个 task 可以并发执行，但若共享可变状态，仍需要同步机制。前一节中的 mutable reference 可作为低层同步工具；更高层并发结构则建立在 task / thread 机制上。

在设计并发程序时，需要区分：

- **并发**：多个操作在时间上交错进行；
- **并行**：多个操作在不同 CPU 资源上同时执行。

Lean 的 task API 同时服务于这两类场景。

## 使用建议

- 若只是把一系列独立工作并行化，优先考虑 task。
- 若共享状态不可避免，优先用高层同步抽象；必须下沉到底层时，再用 `IO.Ref` / `ST.Ref` 配合锁语义。
- 对结果聚合和错误处理，要明确谁负责等待 task、谁负责传播失败。
