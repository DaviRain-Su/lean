# 环境变量

> 对应英文：[Environment Variables](https://lean-lang.org/doc/reference/latest/IO/Environment-Variables/)，抓取日期：2026-06-16。

Lean 提供访问进程环境变量的 API。环境变量是操作系统为进程提供的键值对配置，常用于：

- 查找外部工具；
- 注入 token、路径或 feature flag；
- 控制 CI / 本地开发行为；
- 传递运行时配置。

## 常见用途

在 Lean 程序里，环境变量最常见的用途包括：

- 读取诸如 `HOME`、`PATH`、`TMPDIR` 等系统配置；
- 从 CI 或 shell 注入配置值；
- 在不修改代码的前提下改变程序行为；
- 决定日志级别、缓存目录、外部命令位置等。

## API 语义

环境变量访问属于 `IO`，因为结果依赖外部进程环境，而不是纯逻辑定义的一部分。读取环境变量时，返回值通常是可选的：某变量可能不存在。

这意味着最自然的处理方式往往是：

- 用 `Option` 表达“变量存在/不存在”；
- 给默认值；
- 或在缺失关键变量时报错。

## 使用建议

- 对必须存在的变量，读取后立刻验证并给出清晰错误信息。
- 对可选变量，使用默认值，不要把“未设置”与“空字符串”混淆。
- 证明或测试中，如果依赖环境变量，应通过包装函数或注入配置隔离外部环境。

英文原页给出了 Lean 访问环境变量的具体命令与辅助函数；实际编码时应优先保留“读取结果是 effectful 且可能缺失”这一设计前提。
