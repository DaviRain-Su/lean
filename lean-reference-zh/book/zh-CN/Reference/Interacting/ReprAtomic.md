# `Repr` 的原子类型

> 对应英文：[Atomic Types](https://lean-lang.org/doc/reference/latest/Interacting-with-Lean/#ReprAtom)，抓取日期：2026-06-16。

某些类型在 `Repr` 中可视为 **atomic**，也就是它们在更大表达式里通常不需要额外括号或特殊结构性处理。

英文页用 `ReprAtom` class 表示这一概念。

## 为什么需要 `ReprAtom`

在打印复杂嵌套结构时，系统需要知道：

- 哪些值可以直接内联；
- 哪些值一旦出现在应用位置就必须加括号。

`ReprAtom` 就是用来表达“这个类型的表示可被当作原子项”的。

## 典型场景

若某个类型的表示天然就是一个原子 token 或紧凑名字，例如某些简单枚举或短标识符，那么把它视为 atomic 往往有助于：

- 避免多余括号；
- 让 `#eval` 输出更简洁；
- 让嵌套的 `Repr` 组合更自然。

## 与 `Repr` 的关系

`ReprAtom` 不是替代 `Repr`，而是为其提供更细粒度信息：

- `Repr` 负责整体怎么表示；
- `ReprAtom` 负责告诉系统“它是否可被安全视为原子项”。

## 使用建议

- 只有在某类型的表示稳定、紧凑且不需要额外括号时，才应把它做成 atomic；
- 对复杂递归结构或操作符风格表达式，不要滥用 atomic，否则输出会变得歧义。
