# 数值字面量

> 对应英文：[Numeric Literals](https://lean-lang.org/doc/reference/latest/Terms/Numeric-Literals/)，抓取日期：2026-06-16。

Lean 有两类数值字面量：

- 自然数字面量
- 科学计数字面量

二者都通过 type class 实现重载，因此字面量并不只属于某一个固定类型。

## 自然数字面量

自然数字面量支持多种写法：

- 十进制：`123`
- 二进制：`0b1010`
- 八进制：`0o755`
- 十六进制：`0xFF`

它们内部还允许下划线分组，例如：

- `1_000_000`
- `0x_c0de_cafe`

但不建议写成难读的形式，比如 `1_2__3`。

### `OfNat`

当 Lean 看到自然数字面量 `n` 时，会把它解释成：

- 查找 `OfNat α n`
- 再使用 `OfNat.ofNat`

因此：

- 字面量可用于很多类型；
- 只要提供合适 `OfNat` instance，就能自定义数值字面量行为。

Lean 标准环境里已有 `OfNat Nat n`，所以在没有更多类型信息时，数字默认落到 `Nat`。

## 科学计数字面量

科学计数通过 `OfScientific` 重载，例如：

- `1.23`
- `121e100`

它们不会先经过 `OfNat` 再转换，而是直接用：

- mantissa
- 指数符号
- 十进制指数

来构造目标值。

标准库为 `Float`、`Float32` 等类型提供了 `OfScientific` 实例。

## 与负数的关系

Lean 没有单独的“整数负数字面量”语法。像 `-5` 实际上是：

- 前缀 `Neg.neg`
- 作用于自然数字面量 `5`

因此负号本身也是可重载运算的一部分。

## 字符串、列表与数组字面量

英文页还在本节最后统一提到：

- 字符串字面量
- 列表字面量
- 数组字面量

其中：

- 字符串详见字符串章节；
- 列表和数组分别写成 `[a, b, c]` 与 `#[a, b, c]`；
- 大列表/数组字面量在实现上会做专门处理，以避免生成过于低效的构造链。

## 使用建议

- 想让某自定义数值类型支持 `42` 这类写法时，优先实现 `OfNat`。
- 想让某类型支持 `1.23` 这类写法时，实现 `OfScientific`。
- 调试字面量推断异常时，先看当前上下文有没有足够类型信息。
