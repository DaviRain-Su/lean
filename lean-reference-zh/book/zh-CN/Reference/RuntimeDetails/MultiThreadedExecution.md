# 多线程执行

> 对应英文：[Multi-Threaded Execution](https://lean-lang.org/doc/reference/latest/Run-Time-Code/Multi-Threaded-Execution/)，抓取日期：2026-06-16。

Lean 运行时提供并发与并行执行支持，核心抽象是 **task**。

## task

task 用来表示：

- 一个未来会产生结果的计算；
- 可能与当前线程并发执行的工作；
- 可被等待、组合、调度的并发单元。

Lean 的 task API 让你能在保持高层函数式风格的同时，表达并发流程。

## task manager

Lean runtime 内置 task manager，用来：

- 给 task 分配硬件资源；
- 调度并发/并行执行；
- 管理任务之间的运行时协作。

因此，多线程执行不是“每次都手工创建原始线程”，而是建立在运行时统一调度之上的。

## 为什么单独列为运行时章节

并发执行的正确性和性能都深受运行时影响。例如：

- 多线程引用计数更新比单线程更贵；
- 是否共享对象会影响复制和同步成本；
- 某些任务是否值得拆成并行粒度，取决于调度开销。

## 使用建议

- 并发设计时，优先从 task API 而不是“底层线程对象”思考；
- 对性能敏感场景，要同时考虑任务粒度与共享数据带来的同步成本；
- 若遇到性能反常，记得把引用计数和跨线程共享也纳入分析。
