# 注释

> 对应英文：[Comments](https://lean-lang.org/doc/reference/latest/Source-Files-and-Modules/#The-Lean-Language-Reference--Source-Files-and-Modules--Concrete-Syntax--Comments)，抓取日期：2026-06-16。

Lean 把注释当作“虽然不是空白字符，但行为上等同于空白”的文件片段。主要有两类注释语法。

## 行注释

若 `--` 不属于某个 token，则它会开启一条行注释。从第一个 `-` 到行尾的所有字符都被当作空白。

## 块注释

若 `/-` 不属于某个 token，且后面**没有紧跟** `-`，则它开启一条块注释，直到遇到匹配的 `-/` 为止。

### 块注释可嵌套

Lean 的块注释支持嵌套：

- 遇到新的 `/-` 会增加嵌套层级；
- 只有所有内部块注释都关闭后，外层 `-/` 才真正结束注释。

这让大段临时屏蔽代码或文档更方便。

## 文档注释不是普通注释

- `/--`
- `/-!`

这两种语法开始的是 **documentation**，而不是普通注释。它们同样以 `-/` 结束，也允许内部嵌套块注释，但它们属于单独的语法类别，其合法放置位置由 Lean grammar 决定。

## 使用建议

- 普通说明优先用 `--` 行注释；
- 需要暂时屏蔽大片代码时可用 `/- ... -/`；
- 要为 declaration 提供 hover 文档时，用 `/-- ... -/` 文档注释，而不是普通注释。
