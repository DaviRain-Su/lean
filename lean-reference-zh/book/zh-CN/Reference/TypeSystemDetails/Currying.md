# Currying

> 对应英文：[Currying](https://lean-lang.org/doc/reference/latest/The-Type-System/Functions/#The-Lean-Language-Reference--The-Type-System--Functions--Currying)，抓取日期：2026-06-16。

在 Lean 核心类型论里，每个函数都只接收**一个**参数，并返回**一个**结果。所谓“多参数函数”，只是通过高阶函数嵌套实现的编码方式，这种编码就叫 **currying**。

## 基本思想

一个看起来接收多个参数的函数，实际上是：

1. 先接收第一个参数；
2. 返回一个新的函数；
3. 新函数再接收剩余参数。

因此，多参数函数的表面语法只是一个方便层；elaboration 之后，结果里仍只有单参数函数。

## 为什么重要

理解 currying 能帮助你看清：

- 为什么函数应用本质上总是“左结合”的；
- 为什么部分应用天然成立；
- 为什么某些高阶函数组合看起来如此自然。

例如，一个二参数函数 `f` 实际上可被看成：

```text
f : α → (β → γ)
```

所以只给它第一个参数后，得到的仍是函数值。

## 与 Lean 语法的关系

Lean 的语法让你写：

- 多参数函数定义
- 多参数函数类型
- 多参数函数应用

但这些都只是语法糖。真正的核心表示依旧是单参数函数链。

## 使用建议

- 读高层 Lean 代码时，记住多参数函数最终会编译成嵌套函数；
- 想理解部分应用、组合子和高阶 API 时，先从 currying 视角思考；
- 调试 elaboration 结果或函数类型不匹配时，这一点尤其关键。
